Description

Use an iterativeDisplay object to quickly display graphics wihtin loops. Basically, when calling a graphic display method, this object stores handle to the created graphics object and then update their properties in the sucessive calls (instead of re-creating a new object). As a result, the display is significantly faster.

Creation

Syntax

To create an iterative display, you simply need to use:
id = iterativeDisplay();

Object Functions

The iterativeDisplay object provides the following methods whose name are similar to Matlab graphic function. During the first iteration, which is used to setup the figures, the behavior of such object function is similar to their Matlab instruction counterpart. However, during the other iterations, the iterativeDisplay only some properties of the created graphic objects.

newIteration

This function has to be called at the begining of every iteration execpt the final one (see finalIteration)
Example:
id.newIteration();

finalIteration

This function must be called at the begining of the last iteration.
Example:
id.finalIteration();

desactivateOptimizedDisplay

This function desactivate all the optimization of the iterativeDisplay. As a result, the behavior of all the function revert back to Matlab's original behavior.
Exemple:
id.desactivateOptimizedDisplay();

disableOptimization

This function desactivate the optimization for a specific instruction or a set of instructions. You can pass a string with a single name function name (plot, etc) or a cell array with multiple function name.
Single instruction optimization desactivation:
id.disableOptimization('plot');
Multiples instructions optimization desactivation:
id.disableOptimization({'plot','title'});
See Optimization desactivation example

isOptimized

This function return true if the optimization are enables, false otherwise.
Example:
disp(id.isFinalIteration)

isFinalIteration

This function return true if the iteration is the last one.
Example:
disp(id.isFinalIteration)

setIter

This function set the properies only during the initialization phase (1st iteration) and final iteration.
Example:
id.setIter(h,'size',10);
See the Setting Properties example

setOnce

This function set the properies during every iteration.
Example:
id.setOnce(h,'size',10);
See the Setting Properties example

skip

This function is a place holder that prevent calling a function. It allows skipping the display of some values, for instance to plot a signal only when certains conditions occurs (e.g. every 10 iterations).
You can skip a call to a single function by calling skip with a single string containing the function name to skip.
You may skip several function scall by using a cell array as parameters.
You may skip two sucessive call to the same function (plot for instance) by using a cell array with duplicate function names.
Example:
id.skip('plot');
id.skip({'plot','semilogx'});
id.skip({'plot','plot','semilogy'});
See Dealing with non available data.

Graphic functions

The following functions are only executed during the initialization phase (1st iteration). Nothing happens in the other iterations.
figure axes subplot tiledlayout nexttile
xlabel ylabel zlabel yyaxis hold grid box
The following instruction is only called during the final iteration: legend
For the following functions, the Matlab code is executed during the initialization phase (1st iteration) and only the properties indicated in parenthesis are updated during the next iterations.
  • title (String)
  • plot (XData,YData)
  • plot3 (XData,YData,ZData)
  • semilogx (XData,YData)
  • semilogy (XData,YData)
  • loglog (XData,YData)
  • surf (XData,YData,ZData)
  • mesh (XData,YData,ZData)
  • sgtitle (String)
  • imgshow (CData) : you may want to check this demo.
  • patch (XData,YData,ZData,CData)
The following functions are executed at every iteration : xlim, ylim, zlim.
